AngularJS中几种服务的比较

发表于 2017-01-03   |  

第一个例子Hello world

在Angular中创建服务一般通过以下几种方式

  • Factory
  • Service
  • Provider
  • Constant
  • Value

其中前三种较为常见，来看一下他们的实现方式：

对比分析

service方式:以类似构造函数的形式定义服务，用new关键字创建了一个服务实例。可以在这个构造函数内部用this定义属性和方法，angualr会隐式返回这个this实例.

factory方式:类似于设计模式中工厂模式。返回一个新建对象，这个对象的属性就是你所创建的服务可提供的api

provider方式:先于.config方法前执行，可以看作是可配置的factory方式。唯一可以对config方法进行注入的服务生成方式。控制器可以访问的其$get()函数里返回的属性和方法。需要注意，在向.config方法注入服务时，需要在服务名后加上”~Provider”。

由此可见，在provider方式生成的服务实例可提供更多的扩展配置，在多个应用中可以通过此种方式使用同一个服务。在Angular内部，factory 和 service其实都是通过provider的方式来实现的。

value与constant

除了以上三种方式之外，value和constant也是可以为controller提供注入的服务

这两者的主要区别是，常量constant可以注入到包括配置函数.config()在内的任何地方而且其值无法被改变，而value则不行。

装饰器decorator

decorator的功能参照设计模式的装饰器模式，可以在不修改原有服务(constant服务不能被修饰)代码的基础上，扩展添加新的功能。

小结

不同于控制器实例会被频繁的销毁或创建，我们所创建的所有服务实例都是单例模式，所以服务经常被用于存储应用层级的数据或是提供与DOM无关的一些操作（如异步请求）。针对不同的服务需求选择不同的创建方式。文中如有错误，欢迎讨论。
