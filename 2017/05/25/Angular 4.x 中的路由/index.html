Angular 4.x 中的路由

  • Angular

安装router与导入

安装Angualr Router:

要使用路由，我们需要在 主模块AppModule 中，导入 RouterModule

主模块中使用 forRoot()，子模块中使用 forChild()

常用指令及常见路由配置项目

名称 说明
Routes 核心路由配置项，指明路由与组件之间的对应关系
RouterOutlet 路由视图“占位符”，有具名和不具名两种，在组件模板中表名路由显示的位置
Router 路由执行对象，在组件控制器中通过navigate()等具体方法执行路由
RouterLink 模板中声明路由配置的指令，一般放在a标签中
ActivatedRoute 当前激活的路由对象。可注入组件使组件控制器获取当前路由信息（如路由参数queryParams）

子路由、守护路由和辅助路由

子路由-children

在ProductComponent组件配置中添加 children，便可在页面中呈现子路由

注意，通过子路由功能可以实现 component-less 路由。使用 component-less 路由允许我们将路由组合在一起，并让它们共享路由配置信息和 outlet

此时， /settings/profile 和 /settings/password 路由定义的内容，将显示在 AppComponent 组件（即根组件而不是之前的子组件）的 router-outlet 元素中

守护路由-canActivate/canDeactivate/resolve

  1. canActivate 只有当满足一定条件才可进入目标路由（如检测当前用户是否登录）

  2. canDeactivate 只有当满足一定条件才可离开当前路由（如提示用户是否保存）

  3. resolve路由接口，在获取相关数据(如异步获取后台数据)之后进入路由，以下是官方例子

辅助路由

在一个组件模板中只能有一个router-outlet指令来标明主路由的显示位置(也称为primary unnamed outlet)，这种路由是匿名的outlet.但是可以有多个具名的outlet (named outlets),即辅助路由.每个具名outlet可以拥有自己独立的路由配置和组件。

在路由配置时，需要注明挂载在哪个具名outlet上

另外可以通过primary控制在进入辅助路由的同时，进入特定的主路由

更多信息可以参考named-outlets

总结

Angular的路由功能十分强大，配置灵活丰富，以上只是部分常见配置，仅是冰山一角，更多内容还需要参考官方文档以及在线Demo。如有不详或错误之处，还请指出，共同学习进步。
